Metaclass magic
by John Weachock in /python at May 13, 11:18pm

Of all David Beazley's magic tricks during his talk at PyCon 2018, probably the loudest gasps of horror were when he used identifiers before defining them (unfortunately, the gasps aren't audible in the recording). During Q&A, someone even asked why there wasn't a NameError. As it turns out, the answer was subtly hiding in his slides.

If you dig through the source code for SLY's lex module, you'll find a __prepare__ method sitting in the metaclass. This magic method is used to create the namespace for the new class definition; if undefined, a regular dict is used. This is the secret behind David's trick: SLY uses a custom subclass of dict that overrides the __getitem__ method to instantiate undefined keys as TokenStr objects instead of raising a NameError.

A minimal example:

prepare.py 
class MetaDict(dict):
  def __getitem__(self, key):
    if key[0] != '_' and key not in self:
      print(f'pretending to have key: {key!r}')
      return key # avoid the NameError

    return super().__getitem__(key)


class Meta(type):
  def __prepare__(name, bases):
    return MetaDict()


class Real(metaclass=Meta):
  Hello, world
